# Contributing

Contributions are welcome if they preserve the safety and redaction model of this repository.

## Acceptable contributions

- documentation improvements
- troubleshooting matrix additions
- safer verification checklists
- redaction improvements
- non-destructive read-only analysis ideas

## Not acceptable

- publishing private transcripts
- adding destructive reset instructions
- uploading raw local logs
- exposing machine-specific secrets
- bypassing runtime safeguards

## Style

Keep contributions practical, specific, and operationally safe. Prefer clear tables, checklists, and minimal assumptions.
